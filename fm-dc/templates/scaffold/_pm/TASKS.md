# Tasks

Active work only — Current / Next / Waiting on / Backlog. Shipped work and thinking-behind-it live in [`sessions/`](./sessions/) (per-day files).

**Current items carry a one-line `Why` + `Done-when`** — the agent reads these to stay oriented on what it's executing. Next / Waiting / Backlog don't need them until promoted to Current. Example:

```markdown
- **Build the search filter UI** — `prototypes/search-v3.html`
  - Why: Sandy can't filter results by category; she's working around it manually.
  - Done when: she can pick a category and see filtered results in under 3 seconds.
```

> Review weekly: scan **Waiting on** every Monday — external blockers rot quietly.
